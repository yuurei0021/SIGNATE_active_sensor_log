# Data Validation Experiment

## 目的
全trainとtestデータがモデルに入力可能な状態かを検証し、前処理の必要性を判断する。

## 検証内容
- train_master.csvの整合性確認
- 全trainデータ（4,523ファイル）の検証
- 全testデータ（1,939ファイル）の検証

### 検証項目
- カラム名が`accelerometer_X`, `accelerometer_Y`, `accelerometer_Z`であること
- 各ファイルが30行であること
- 欠損値（NaN）がないこと
- すべての値が数値型であること
- 無限大値がないこと

## 結果

### train_master.csv
- 件数: 4,523件
- クラス分布:
  - running: 2,361件 (52.2%)
  - walking: 1,311件 (29.0%)
  - idle: 741件 (16.4%)
  - stairs: 110件 (2.4%)
- 欠損値: なし

### trainデータ
- ファイル数: 4,523
- 有効ファイル: 4,523/4,523 (100%)
- 無効ファイル: 0

### testデータ
- ファイル数: 1,939
- 有効ファイル: 1,939/1,939 (100%)
- 無効ファイル: 0

## 結論
**全データがモデルに入力可能な状態であることを確認。前処理は不要。**

## 備考
- クラス不均衡あり（stairs: 2.4%）- 層化サンプリングやクラス重み付けを検討
- データ形状は統一されており、直接モデリング可能
